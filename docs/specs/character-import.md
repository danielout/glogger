# Character Report Import — Feature Spec

## Purpose

Import character report JSON files generated by the game's `/outputcharacter` command. These reports contain a complete snapshot of a character's skills, stats, favor, currencies, quests, and recipe completions at a point in time. By importing snapshots over time, players can track character progression.

This spec covers the **core character import** only. Inventory import (`/outputitems`) is a separate feature.

---

## Character Report File Format

### File Naming & Location
- Pattern: `Character_{CharacterName}_{ServerName}.json`
- Location: `{game_data_path}/` (same root as chat logs)
- The file is overwritten each time `/outputcharacter` is run — there is only ever one file per character+server

### Top-Level JSON Structure

| Field | Type | Description |
|-------|------|-------------|
| `Character` | `string` | Character name |
| `ServerName` | `string` | Server name (e.g. "Dreva") |
| `Timestamp` | `string` | ISO-ish format: `"2026-03-08 14:56:31Z"` |
| `Report` | `string` | Always `"CharacterSheet"` for this report type |
| `ReportVersion` | `number` | Schema version from the game (currently `1`) |
| `Race` | `string` | Character race (e.g. "Human") |
| `Skills` | `object` | Skill name → skill data (see below) |
| `RecipeCompletions` | `object` | Recipe internal name → completion count |
| `CurrentStats` | `object` | Stat key → numeric value |
| `Currencies` | `object` | Currency key → amount |
| `ActiveQuests` | `string[]` | List of active quest internal names |
| `ActiveWorkOrders` | `string[]` | List of active work order internal names |
| `CompletedWorkOrders` | `string[]` | List of completed work order internal names |
| `NPCs` | `object` | NPC key → `{ "FavorLevel": string }` |

### Skills Object

Each key in `Skills` is a skill name (e.g. `"Hammer"`, `"Anatomy_Goblins"`). The value:

```json
{
    "Level": 79,
    "BonusLevels": 6,
    "XpTowardNextLevel": 251229,
    "XpNeededForNextLevel": 1703383,
    "Abilities": ["Pound1", "RecklessSlam1", ...]
}
```

| Field | Type | Description |
|-------|------|-------------|
| `Level` | `number` | Current skill level |
| `BonusLevels` | `number` | Bonus levels from gear/buffs |
| `XpTowardNextLevel` | `number` | XP progress toward next level |
| `XpNeededForNextLevel` | `number` | Total XP required for next level (-1 means max level) |
| `Abilities` | `string[]?` | Optional. List of ability internal names learned for this skill |

### NPCs Object

Each key is an NPC internal key (e.g. `"NPC_Ashk"`, `"Cow_Bessie"`). The value:

```json
{ "FavorLevel": "SoulMates" }
```

Known favor levels: `Neutral`, `Comfortable`, `Friends`, `CloseFriends`, `BestFriends`, `LikeFamily`, `SoulMates`

### CurrentStats Object

Flat key-value pairs of stat identifiers to numeric values:

```json
{
    "MAX_HEALTH": 590,
    "MAX_ARMOR": 641,
    "MAX_POWER": 446,
    "MOVEMENT_SPEED": 5,
    "MAX_INVENTORY_SIZE": 142,
    ...
}
```

### Currencies Object

```json
{
    "GOLD": 68262,
    "GUILDCREDITS": 10,
    "COMBAT_WISDOM": 10563,
    ...
}
```

## Database Tables

Existing tables from the V1 schema are noted below. New tables (marked **NEW TABLE**) need to be added to the migration in [migrations.rs](../../src-tauri/src/db/migrations.rs). Since we're in early prototyping, add them directly to the V1 migration (rebuild from scratch).

### `character_snapshots`
One row per import. Stores the character identity, timestamp, and full raw JSON.

| Column | Source |
|--------|--------|
| `character_name` | `Character` |
| `server_name` | `ServerName` |
| `snapshot_timestamp` | `Timestamp` |
| `race` | `Race` |
| `raw_json` | Entire file contents |

**Deduplication:** UNIQUE constraint on `(character_name, server_name, snapshot_timestamp)`. Re-importing the same file is a no-op (upsert or skip).

### `character_skill_levels`
One row per skill per snapshot. Abilities are **not** stored here — they are derivable from the CDN data and skill level.

| Column | Source |
|--------|--------|
| `snapshot_id` | FK to `character_snapshots.id` |
| `skill_name` | Key from `Skills` object |
| `level` | `Level` |
| `bonus_levels` | `BonusLevels` |
| `xp_toward_next` | `XpTowardNextLevel` |
| `xp_needed_for_next` | `XpNeededForNextLevel` |

### `character_npc_favor`
One row per NPC per snapshot.

| Column | Source |
|--------|--------|
| `snapshot_id` | FK to `character_snapshots.id` |
| `npc_key` | Key from `NPCs` object |
| `favor_level` | `FavorLevel` value |

### `character_recipe_completions` (NEW TABLE)
One row per recipe per snapshot. Tracks how many times each recipe has been completed.

| Column | Type | Source |
|--------|------|--------|
| `id` | `INTEGER PRIMARY KEY` | Auto |
| `snapshot_id` | `INTEGER NOT NULL` | FK to `character_snapshots.id` |
| `recipe_key` | `TEXT NOT NULL` | Key from `RecipeCompletions` object (e.g. `"Butter"`, `"FireMagic0"`) |
| `completions` | `INTEGER NOT NULL` | Completion count |

Indexes: `(snapshot_id)`, `(recipe_key, snapshot_id)`

### `character_stats` (NEW TABLE)
One row per stat per snapshot. Stores the character's computed stats at time of export.

| Column | Type | Source |
|--------|------|--------|
| `id` | `INTEGER PRIMARY KEY` | Auto |
| `snapshot_id` | `INTEGER NOT NULL` | FK to `character_snapshots.id` |
| `stat_key` | `TEXT NOT NULL` | Key from `CurrentStats` (e.g. `"MAX_HEALTH"`, `"MOVEMENT_SPEED"`) |
| `value` | `REAL NOT NULL` | Stat value (REAL to handle both integer and decimal stats) |

Indexes: `(snapshot_id)`, `(stat_key, snapshot_id)`

### `character_currencies` (NEW TABLE)
One row per currency per snapshot.

| Column | Type | Source |
|--------|------|--------|
| `id` | `INTEGER PRIMARY KEY` | Auto |
| `snapshot_id` | `INTEGER NOT NULL` | FK to `character_snapshots.id` |
| `currency_key` | `TEXT NOT NULL` | Key from `Currencies` (e.g. `"GOLD"`, `"COMBAT_WISDOM"`) |
| `amount` | `INTEGER NOT NULL` | Currency amount |

Indexes: `(snapshot_id)`, `(currency_key, snapshot_id)`

### What is NOT stored in normalized tables

The following data lives only in `raw_json`. It can be normalized in a future migration:

- `ActiveQuests` / `ActiveWorkOrders` / `CompletedWorkOrders` — quest/work order lists
- `Abilities` lists within each skill — derivable from CDN data + skill level

## Rust Implementation

### Module: `src-tauri/src/db/character_commands.rs`

#### JSON Deserialization Structs

```rust
#[derive(Deserialize)]
#[serde(rename_all = "PascalCase")]
pub struct CharacterReport {
    pub character: String,
    pub server_name: String,
    pub timestamp: String,
    pub report: String,          // "CharacterSheet"
    pub report_version: u32,
    pub race: String,
    pub skills: HashMap<String, SkillData>,
    pub recipe_completions: HashMap<String, i64>,
    pub current_stats: HashMap<String, f64>,
    pub currencies: HashMap<String, i64>,
    #[serde(rename = "NPCs")]
    pub npcs: HashMap<String, NpcFavorData>,
    // ActiveQuests, WorkOrders etc. captured only in raw_json
}

#[derive(Deserialize)]
#[serde(rename_all = "PascalCase")]
pub struct SkillData {
    pub level: i32,
    pub bonus_levels: i32,
    pub xp_toward_next_level: i64,
    pub xp_needed_for_next_level: i64,
    pub abilities: Option<Vec<String>>,
}

#[derive(Deserialize)]
#[serde(rename_all = "PascalCase")]
pub struct NpcFavorData {
    pub favor_level: String,
}
```

#### Commands

**`import_character_report`** — Primary import command

- **Input:** `file_path: String` (path to the JSON file)
- **Flow:**
  1. Read the file contents into a string
  2. Deserialize into `CharacterReport`
  3. Validate `report` field equals `"CharacterSheet"`
  4. Begin a database transaction
  5. INSERT into `character_snapshots` with `ON CONFLICT ... DO NOTHING` (skip duplicates)
  6. If the insert created a new row, get the `last_insert_rowid()`
  7. Batch INSERT skill levels into `character_skill_levels`
  8. Batch INSERT NPC favor into `character_npc_favor`
  9. Batch INSERT recipe completions into `character_recipe_completions`
  10. Batch INSERT stats into `character_stats`
  11. Batch INSERT currencies into `character_currencies`
  12. Commit transaction
- **Output:** `Result<ImportResult, String>`

```rust
#[derive(Serialize)]
pub struct ImportResult {
    pub character_name: String,
    pub server_name: String,
    pub snapshot_timestamp: String,
    pub skills_imported: usize,
    pub npcs_imported: usize,
    pub recipes_imported: usize,
    pub stats_imported: usize,
    pub currencies_imported: usize,
    pub was_duplicate: bool,
}
```

**`get_character_snapshots`** — List all snapshots for a character

- **Input:** `character_name: String`, `server_name: Option<String>`
- **Output:** `Vec<CharacterSnapshotSummary>` ordered by `snapshot_timestamp DESC`

```rust
#[derive(Serialize)]
pub struct CharacterSnapshotSummary {
    pub id: i64,
    pub character_name: String,
    pub server_name: String,
    pub snapshot_timestamp: String,
    pub race: String,
    pub import_date: String,
    pub skill_count: usize,
}
```

**`get_snapshot_skills`** — Get all skill levels for a specific snapshot

- **Input:** `snapshot_id: i64`
- **Output:** `Vec<SnapshotSkillLevel>`

```rust
#[derive(Serialize)]
pub struct SnapshotSkillLevel {
    pub skill_name: String,
    pub level: i32,
    pub bonus_levels: i32,
    pub xp_toward_next: i64,
    pub xp_needed_for_next: i64,
}
```

**`get_snapshot_npc_favor`** — Get all NPC favor for a specific snapshot

- **Input:** `snapshot_id: i64`
- **Output:** `Vec<SnapshotNpcFavor>`

**`get_snapshot_recipes`** — Get all recipe completions for a specific snapshot

- **Input:** `snapshot_id: i64`
- **Output:** `Vec<SnapshotRecipeCompletion>`

```rust
#[derive(Serialize)]
pub struct SnapshotRecipeCompletion {
    pub recipe_key: String,
    pub completions: i64,
}
```

**`get_snapshot_stats`** — Get all stats for a specific snapshot

- **Input:** `snapshot_id: i64`
- **Output:** `Vec<SnapshotStat>`

```rust

    pub stat_key: String,
    pub value: f64,
}
```

**`get_snapshot_currencies`** — Get all currencies for a specific snapshot

- **Input:** `snapshot_id: i64`
- **Output:** `Vec<SnapshotCurrency>`

```rust
#[derive(Serialize)]
pub struct SnapshotCurrency {
    pub currency_key: String,
    pub amount: i64,

```

**`get_characters`** — List all known characters

- **Output:** `Vec<CharacterInfo>` (distinct character_name + server_name pairs with latest snapshot date)

**`compare_snapshots`** — Compare skills between two snapshots

- **Input:** `snapshot_id_old: i64`, `snapshot_id_new: i64`
- **Output:** `Vec<SkillDiff>` showing level changes, XP changes

```rust
#[derive(Serialize)]
pub struct SkillDiff {
    pub skill_name: String,
    pub old_level: i32,
    pub new_level: i32,
    pub level_change: i32,
    pub old_xp: i64,
    pub new_xp: i64,
}
```

## Edge Cases & Validation

| Case | Handling |
|------|----------|
| Duplicate import (same char+server+timestamp) | Skip via UNIQUE constraint, return `was_duplicate: true` |
| Missing `Abilities` field on a skill | Optional field — just skip, not stored |
| `XpNeededForNextLevel: -1` | Store as-is. Means skill is at max level |
| `Report` field is not `"CharacterSheet"` | Return error — wrong report type |
| Malformed JSON | Return descriptive parse error |
| Empty `Skills` or `NPCs` object | Valid — import with 0 skills/NPCs |
| Recipe with 0 completions | Store it — useful for tracking which recipes are known |
| Stat values that are floats (e.g. `1.14`) | Stored as REAL — handles both int and decimal |
| Currency amount of 0 | Store it — shows the currency exists for this character |
| NPC keys that don't match the `npcs` table | Store anyway — `npc_key` is just a text field, no FK enforced |
